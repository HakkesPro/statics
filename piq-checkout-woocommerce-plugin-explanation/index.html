Wordpress PaymentIQ-checkout plugin explained step-by-step

Introduction:

The PaymentIQ checkout plugin is a combination of our cashier and checkout client being integrated via bootstrapper script as described in https://docs.paymentiq.io/europe/cashier/general/initialize.

We also include a “lookupConfig” parameter to that script to include the checkout flow (where you identify yourself/collect user details).

But from an integration point of view, you only need that bootstrapper script and add some parameters to that as described in docs above, simple as that.

This wordpress plugin is desbribed more technically in confluence -> https://confluence.techno.ingenico.com/pages/viewpage.action?pageId=220288723#PaymentIQCheckout-woocommerce-plugin.

  1. Download the plugin via wordpress-admin plugins.

We distribute this plugin to wordpress via wordpress official plugin SVN repository and that makes it accessible when you search for “Paymentiq checkout” in wordpress admin -> plugins.

  1. The plugin is accessible in WP-admin in plugins section

After downloading the plugin from WP-admin it’s now accessible in WP-admin plugins view. You also need to repeat this process when downloading the woocommerce plugin as this plugin is used inside woocommerce plugin.

  1. Click on “Activate” for both “PaymentIQ checkout” and “Woocommerce” plugin and then select “PaymentIQ checkout” as “Payment” in woocommerce settings and click on “PaymentIQ Checkout” to enter settings page for the checkout.

  1. Add PIQ and cashier settings in PaymentIQ checkout (MID, UID, theme options etc..)

  1. Our plugin content is first visible at the “Product”, and “Cart” page if “Enable loan calculator” is enabled. (optional)

This should take the product price as a parameter and will calculate Santander partially payment prices and monthly fees for the user.

This is integrated via a script tag and a html tag and a class you trigger with configs as parameters.

It will render like this on the Cart page.

  1. “Proceed to checkout” and you will see our Checkout-Client first then after entering your user information it will display our cashier

These applications are bootstrapped easily via a script tag, html tag, and triggering a class with configs as parameters. (Similar to the calculator widget). (https://docs.paymentiq.io/europe/cashier/general/initialize)

  1. Order history is viewed via the WP-admin woocommerce “Orders”. We use some admin API features from here..

One should be able from this view to handle orders, making refunds, cancellations and captures using PaymentIQ’s admin api (https://docs.paymentiq.io/europe/admin/general/introduction).
